## 1. Project Proposal

### 1.1 Project Title
CraveBites — Online Food Ordering Web Application

### 1.2 Objective
Design and develop a full-stack, database-driven web application for an online food business. The system allows customers to browse products and deals, register accounts, add items to a cart, and place delivery orders. A separate admin panel allows authorised staff to manage all site content including products, deals, and customer enquiries.

CraveBites is a food delivery brand based in Colombo, Sri Lanka. The website targets local customers who want to browse the menu, discover combo deals, and order online. All prices are displayed in Sri Lankan Rupees (LKR).

## 3. System Implementation

### 3.1 Public Modules

**Authentication Module (`signup.php`, `signin.php`, `signout.php`)**
- Customers register with name, email, and password (hashed with `password_hash(PASSWORD_DEFAULT)`)
- Login validated using `password_verify()`
- Session keys: `$_SESSION['user_id']`, `user_name`, `user_email`
- `require_user()` helper redirects unauthenticated users to Sign In

**Product Listing Module (`products.php`)**
- Fetches all products from the `products` table grouped by category
- JavaScript filter tabs switch visible category sections
- Each card has an inline POST form that calls `cart.php?action=add`

**Deals Module (`deals.php`)**
- Fetches records from `deals` WHERE `is_active = 1`
- Displays tag badge, short note, item list (newline-split), deal price, and original price in LKR

**Cart Module (`cart.php`)**
- Handles POST actions: `add`, `update`, `remove`, `clear`
- `INSERT … ON DUPLICATE KEY UPDATE` prevents duplicate rows per user/product
- Displays running subtotal; links to Checkout

**Checkout Module (`checkout.php`)**
- Three delivery options: **Standard** (LKR 350, 2–3 hrs), **Express** (LKR 700, 45–60 min), **Pickup** (free)
- Uses a PDO transaction to insert into `orders` and `order_items` atomically
- Clears `cart_items` for the user on successful order placement

**Contact Form (`contact.php`)**
- Validates name, email, and message server-side
- Inserts a new row into the `contacts` table

### 3.2 Admin Modules

**Admin Authentication (`admin/login.php`, `admin/logout.php`)**
- Separate session key `$_SESSION['admin_logged_in']`
- All admin pages check this key at the top and redirect to login if not set

**Product CRUD**
- `admin/products.php` — SELECT all products, DELETE by ID
- `admin/product_form.php` — INSERT new product or UPDATE existing by ID
- Price input accepted in LKR and stored in base units

**Deals CRUD**
- `admin/deals.php` — SELECT all deals, toggle active status, DELETE by ID
- `admin/deal_form.php` — INSERT new deal or UPDATE existing; `is_active` checkbox

**Contact Message Management**
- `admin/contacts.php` — SELECT all messages; DELETE by ID

### 3.3 Database Integration
- PDO with `PDO::ERRMODE_EXCEPTION` used for all database access (`config/database.php`)
- Prepared statements with bound parameters used for all INSERT, UPDATE, and DELETE to prevent SQL injection
- `cart_items` and `order_items` use foreign key constraints with `ON DELETE CASCADE`
- Shared helpers in `includes/functions.php`: `url()`, `format_price()`, `lkr_rate()`, `to_base_from_lkr()`, `cart_item_count()`, `is_user_logged_in()`, `current_user_id()`, `current_user_name()`

## 4. Database Design Details

### 4.1 Entity Relationship Summary

```
users ──< cart_items >── products
users ──< orders ──< order_items >── products
contacts (standalone)
deals (standalone)
news (legacy, admin only)
```

### 4.2 Table Definitions

#### `products`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| name | VARCHAR(120) | Product name |
| category | VARCHAR(80) | e.g. Main Courses, Sides, Beverages and Drinks |
| description | TEXT | Product description |
| price | DECIMAL(10,2) | Base price (multiply by LKR rate for display) |
| image_url | VARCHAR(300) | Unsplash image URL |
| created_at | TIMESTAMP | Auto-set on insert |

#### `deals`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| title | VARCHAR(150) | Deal name |
| tag | VARCHAR(60) | Badge label (e.g. "Best Value") |
| short_note | VARCHAR(255) | One-line description |
| items_text | TEXT | Newline-separated item list |
| deal_price_lkr | DECIMAL(10,2) | Discounted price in LKR |
| original_price_lkr | DECIMAL(10,2) | Original price in LKR |
| image_url | VARCHAR(300) | Cover image URL |
| is_active | TINYINT(1) | 1 = visible on site, 0 = hidden |
| created_at | TIMESTAMP | Auto-set on insert |

#### `contacts`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| full_name | VARCHAR(120) | Sender's name |
| email | VARCHAR(180) | Sender's email |
| message | TEXT | Message body |
| submitted_at | TIMESTAMP | Auto-set on insert |

#### `users`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| full_name | VARCHAR(120) | Customer name |
| email | VARCHAR(180) UNIQUE | Login email |
| password_hash | VARCHAR(255) | bcrypt hash via `password_hash()` |
| created_at | TIMESTAMP | Auto-set on insert |

#### `cart_items`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| user_id | INT | FK → `users.id` CASCADE DELETE |
| product_id | INT | FK → `products.id` CASCADE DELETE |
| quantity | INT | Minimum 1 |
| created_at | TIMESTAMP | Auto-set on insert |
| — | UNIQUE | `(user_id, product_id)` |

#### `orders`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| user_id | INT | FK → `users.id` CASCADE DELETE |
| full_name | VARCHAR(120) | Delivery recipient |
| phone | VARCHAR(40) | Contact number |
| address_line | VARCHAR(255) | Street address |
| city | VARCHAR(120) | Delivery city |
| delivery_type | VARCHAR(40) | `standard` / `express` / `pickup` |
| delivery_fee_lkr | DECIMAL(10,2) | 0 / 350 / 700 |
| subtotal_lkr | DECIMAL(10,2) | Sum of line totals |
| total_lkr | DECIMAL(10,2) | subtotal + delivery fee |
| status | VARCHAR(40) | Default: `Pending` |
| created_at | TIMESTAMP | Auto-set on insert |

#### `order_items`
| Column | Type | Notes |
|---|---|---|
| id | INT AUTO_INCREMENT | Primary Key |
| order_id | INT | FK → `orders.id` CASCADE DELETE |
| product_id | INT | FK → `products.id` RESTRICT |
| product_name | VARCHAR(140) | Snapshot at order time |
| unit_price_lkr | DECIMAL(10,2) | Price at order time |
| quantity | INT | Units ordered |
| line_total_lkr | DECIMAL(10,2) | unit_price × quantity |

### 4.3 DML Operations Demonstrated

| Operation | Where Used |
|---|---|
| `INSERT` | New products, deals, orders, order_items, cart_items, contacts, user registration |
| `SELECT` | All public listing pages, cart page, checkout, admin tables, dashboard counts |
| `UPDATE` | Edit product, edit deal, update cart item quantity |
| `DELETE` | Remove product, remove deal, delete contact message, remove cart item, clear cart |
